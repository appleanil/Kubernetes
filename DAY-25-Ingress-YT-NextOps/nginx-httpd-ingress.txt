Real-Time DevOps Interview Answer

Ingress is a Kubernetes resource used to expose multiple applications externally using a single LoadBalancer/IP and route traffic based on path or hostname.
It helps reduce cloud cost and centralize traffic management using an Ingress Controller.

Kubernetes Ingress — Detailed Explanation

In Kubernetes, Ingress is used to expose HTTP/HTTPS applications from outside the cluster and route traffic to different services.

Problem Before Ingress

Suppose you have 3 applications:
NGINX app
HTTPD app
Spring Boot app

Without Ingress:
NGINX   -> LoadBalancer
HTTPD   -> LoadBalancer
SPRING  -> LoadBalancer

Each service needs:
Separate external IP
Separate AWS LoadBalancer
Higher cost
Solution Using Ingress

With Ingress:
Internet
    ↓
Single LoadBalancer
    ↓
Ingress Controller
    ↓
Routes traffic to services


Now:
One LoadBalancer
One external IP
Multiple applications
Real-Time Architecture


User Request
     ↓
AWS LoadBalancer
     ↓
NGINX Ingress Controller
     ↓
Ingress Rules
     ↓
Service
     ↓
   Pods


Main Components
----------------
| Component          | Purpose               |
| ------------------ | --------------------- |
| Ingress            | Defines routing rules |
| Ingress Controller | Executes rules        |
| Service            | Connects to pods      |
| Pod                | Runs application      |


What Ingress Can Do
1. Path-Based Routing
Example:
/nginx  -> nginx-service
/httpd  -> httpd-service
User accesses:
http://EXTERNAL-IP/nginx
http://EXTERNAL-IP/httpd

2. Host-Based Routing
Example:
app1.company.com -> nginx-service
app2.company.com -> spring-service

3. SSL/HTTPS Termination
Ingress can manage:
SSL certificates
HTTPS traffic
Example:
https://myapp.com

4. Load Balancing
Ingress distributes traffic across:
Multiple pods
Multiple nodes

5. URL Rewrite
Example:
/app1/test -> /
using annotations.

Example Ingress YAML
apiVersion: networking.k8s.io/v1
kind: Ingress

metadata:
  name: my-ingress

spec:
  ingressClassName: nginx

  rules:
  - http:
      paths:

      - path: /nginx
        pathType: Prefix
        backend:
          service:
            name: nginx-service
            port:
              number: 80

https://raw.githubusercontent.com/kubernetes/ingress-nginx/controller-v1.2.1/deploy/static/provider/cloud/deploy.yaml 
firstly apply this yaml file

kubectl apply -f ingress-controller.yaml

apiVersion: networking.k8s.io/v1
kind: Ingress

metadata:
  name: k8s-ingress
  namespace: prod

  annotations:
    nginx.ingress.kubernetes.io/ssl-redirect: "false"
    nginx.ingress.kubernetes.io/use-regex: "true"
    nginx.ingress.kubernetes.io/rewrite-target: /$2

spec:
  ingressClassName: nginx

  rules:
  - http:
      paths:

      # nginx app
      - path: /nginx(/|$)(.*)
        pathType: ImplementationSpecific
        backend:
          service:
            name: nginx
            port:
              number: 80

      # httpd app
      - path: /httpd(/|$)(.*)
        pathType: ImplementationSpecific
        backend:
          service:
            name: httpd
            port:
              number: 80

      # default backend
      - path: /(.*)
        pathType: ImplementationSpecific
        backend:
          service:
            name: nginx
            port:
              number: 80
