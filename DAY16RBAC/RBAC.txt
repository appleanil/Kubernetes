RBAC
====
RBAC (Role-Based Access Control) in Kubernetes is a powerful method for regulating access to the Kubernetes API based on the roles of individual users or groups.
Here's a breakdown of its key components and how it works:

Key Components
--------------
ROLE: Defines a set of permissions within a specific namespace. It grants access to resources like pods, deployments, etc.
1. Role (రోల్) ----------> Kubernetes లో ఒక specific **namespace** లో ఏఏ permissions ఉండాలో define చేస్తుంది.
Example:
* pods చూడటం
* deployments create చేయటం
* services update చేయటం
అంటే, ఒక namespace లో user ఏమి చేయగలడో Role నిర్ణయిస్తుంది.

2.RoleBinding: Grants the permissions defined in a Role to a user or a group within a specific namespace.
2. RoleBinding (రోల్ బైండింగ్) ------> Role లో define చేసిన permissions ను ఒక **user**, **group**, లేదా **service account** కు assign చేస్తుంది.
అంటే:* “ఈ user కి ఈ Role permissions ఇవ్వాలి” అని connect చేసే component.
ఇది కూడా ఒక specific namespace లో మాత్రమే పనిచేస్తుంది.

3. ClusterRole: Similar to a Role but applies across the entire cluster. It can be used for non-namespaced resources or for cluster-wide permissions.
3. ClusterRole (క్లస్టర్ రోల్) --------> ఇది Role లాంటిదే కానీ మొత్తం **cluster** కి వర్తిస్తుంది.
ఇది ఉపయోగించే సందర్భాలు:
* cluster-wide permissions ఇవ్వాలి
* nodes వంటి non-namespaced resources access చేయాలి
Example:
* అన్ని namespaces లో pods చూడటం
* nodes access చేయటం

4. ClusterRoleBinding: Grants the permissions defined in a ClusterRole to a user or a group across the entire cluster.
4. ClusterRoleBinding -------> ClusterRole లో ఉన్న permissions ను ఒక **user**, **group**, లేదా **service account** కు cluster మొత్తం మీద assign చేస్తుంది.

అంటే:* “ఈ user కి cluster అంతటా ఈ permissions ఇవ్వాలి” అని bind చేస్తుంది.


| Role + RoleBinding                              | ClusterRole + ClusterRoleBinding                             |
| ----------------------------------------------- | ------------------------------------------------------------ |
| Specific namespace లో మాత్రమే పనిచేస్తుంది        | మొత్తం cluster లో పనిచేస్తుంది                                 |
| Example: `dev` namespace                        | Example: అన్ని namespaces                                    |
| Role → permissions define చేస్తుంది               | ClusterRole → cluster-wide permissions define చేస్తుంది        |
| RoleBinding → Role ని user కి assign చేస్తుంది     | ClusterRoleBinding → ClusterRole ని user కి assign చేస్తుంది     |
| Namespace level access control                  | Cluster level access control                                 |
| Example Permission: pod create                  | Example Permission: logs view across cluster                 |
| Only `dev` namespace లో pod create చేయగలరు    | అన్ని namespaces లో logs చూడగలరు                           |
| Namespaced resources కోసం ఎక్కువగా ఉపయోగిస్తారు  | Cluster-wide / non-namespaced resources కోసం ఉపయోగిస్తారు    |



Short Difference Table
----------------------

| Component          | Scope            | Purpose                             |
| ------------------ | ---------------- | ----------------------------------- |
| Role               | Single Namespace | Namespace permissions               |
| RoleBinding        | Single Namespace | Role ని user కి assign చేయటం       |
| ClusterRole        | Entire Cluster   | Cluster-wide permissions            |
| ClusterRoleBinding | Entire Cluster   | ClusterRole ని user కి assign చేయటం|
