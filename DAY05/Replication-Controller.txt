ReplicationController
=====================
--> Replication Controller is one of the key features of Kubernetes, which is responsible for managing the pod lifecycle.
    And make sure that the specified no:of pod replicas are running at any point of time.

--> A Replication Controller is a structure that enables you to easily create multiple pods, then make sure that number of pods always exists. 
    If a pod does crash, the Replication Controller replaces it.
    If a pod does del, the Replication Controller recreate it.

--> Replication Controllers and PODS are associated with labels.

--> Creating “RC” with count of 1 ensure that a POD is always available.

--> when they ask down the application then dnt delete the RC. Do replicas scale zero ---> kubectl scale rc <rc-name> --replicas=0

--> When u want to expose the internet then write yaml file for service with NodePort

--> In ReplicationController selector is optional. If not specified, Kubernetes automatically uses the labels defined in the Pod template.



Example:
--------

apiVersion: v1
kind: ReplicationController
metadata:
  name: javawebapprc
  namespace: test-ns
  labels:
    rcapp: javaapprc
    env: prod # we can define no:of labels
spec: # replicationcontroller spec from replicas to app
  replicas: 2
  selector:
    app: javawebapp # here app: javawebapp same in temp meta app:javawebapp
  template:
    metadata: 
      name: javawebapprcpod
      labels:
        app: javawebapp
        env: prod
        team: devops
    spec: #pod spec
      containers:
      - name: javawebapprccon
        image: kkeducationb2/java-webapp:1.1 #anildocker11/springwebapp:0.0.2 or 0.0.1 / anildocker11/maven:0.0.1 / 0.0.2 / 0.0.3 or anildocker11/docker-maven:0.0.0
        ports:
        - containerPort: 8080

save: rc.yaml

kubectl apply -f rc.yaml --dry-run=client

kubectl apply -f rc.yaml --dry-run=server

kubectl apply -f rc.yaml

kubectl get rc -n <NS>

kubectl get all -n <NS>

kubectl scale rc <rc_Name> --replicas 3 -n <NS>

kubectl delete rc <rc_Name> -n <NS>
